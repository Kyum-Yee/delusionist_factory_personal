# Delusionist Factory - One-Click Prompt

아래 내용을 AI 에이전트(Claude, Gemini 등)에게 전달하면 자동으로 전체 파이프라인을 실행합니다.

---

## 🚀 How to Use

1. `input/request.json`에 원하는 주제 설정
2. 이 prompt.txt 내용을 AI에게 붙여넣기
3. AI가 알아서 Step 1 → Step 2 → Step 3 완료

---

## 📋 One-Click Request Prompt

```
/Users/jakesmacair/프로젝트 파일/delusionist_factory 에서 Delusionist Factory를 실행해줘.

작업 내용:
1. input/request.json 읽기
1.5. (중요) /Users/jakesmacair/프로젝트 파일/.claude/skills/delusionist/SKILL.md 파일을 반드시 읽고 메모리에 저장한 뒤, 가이드에 따르기.
2. main.py 실행 → 출력된 Agent 작업 지시 확인
3. Agent 작업 수행 (망상적 변이 문장 생성)
4. 결과를 output/ 폴더의 해당 파일에 append
5. 다시 main.py 실행 → 다음 배치 또는 다음 Step 진행
6. "ALL STEPS COMPLETE" 메시지가 나올 때까지 반복

주의사항:
- Step 1: section_a_chains.txt에 CHAINS_COUNT개의 문장 생성 (넘버링 + 무작위 단어 **볼드** 표시 + 문장 끝 괄호 주석 필수)
- Step 2: section_b_refined.txt에 SELECTION_B_COUNT개의 정제 문장 생성 (각 문장 끝에 괄호로 짧은 주석 필수 — 인상적인 점, C단계 활용 방향 메모. Step 1 주석은 선택적 참고)
- Step 3: section_c_final.txt에 REFINING_COUNT개의 최종 결과물 생성 (Step 2 주석은 선택적 참고)
- 매 문장에 MANDATORY_WORD 필수 포함
- 한국어+영어 혼용시 영어 3단어 연속 사용 금지

끝나면 output/section_c_final.txt 내용을 보여줘.
```

## 🔀 Step 1 병렬 실행 (빠르게 돌리기)

Step 1은 sub-agent를 사용해 병렬로 실행할 수 있다. 순서 무관 — 총 갯수만 맞으면 됨.

```
1. prepare_parallel_step1 호출 (두 가지 모드, batch_size 우선):
   - batch_size=25 → 워커당 25줄, 워커 수 자동 계산 (예: 50줄 남음 → 2워커)
   - worker_count=3 → 3워커, 줄 수 균등 분할
2. 반환된 워커별 random_words + context를 각 sub-agent에 전달
3. 각 sub-agent가 문장 생성 후 append_result(step="1") 호출
4. 실패한 워커 있으면 → get_status로 부족분 확인 → prepare_parallel_step1(worker_count=1)로 재시도
5. 완료 후 run_delusionist → Step 2 진입
```

동시 쓰기는 파일 잠금(fcntl.flock)으로 보호됨.

## ⚠️ DIRECTION 설정 원칙 : 아래 모든 내용을 본인만의 언어로 표현

### 1. DIRECTION의 범위

**포함: 구조적 요소**
형식과 논리 골격은 자유롭게 명시한다.
- 형식: 서정시, 에세이, 논설, 블로그 등
- 논리구조: "왜 되는지", "어떻게 따라하는지" 등 설명 프레임
- 챕터/섹션 구성: 도입-전개-결론, 단계별 가이드 등
- 좋은 예: "서정시 한 편 창작" (형식만 제시 → 주제와 감성은 체이닝에서 발견)

**제외: 상상의 폭**
구체적 분위기, 이미지어, 감성 방향은 넣지 않는다.
- 나쁜 예: "비 오는 밤 창가에서 기억을 회상하는 시" (발산 불가)
- 유저가 명시적으로 요청한 경우에만 포함한다.
- 미지정 감성/분위기는 Step 1~2의 확률적 오염에서 자연 발생하도록 열어둔다.

**유저 명시 시: 감성·이성 프레임워크**
문학·형이상학 같은 감성 영역, 그림체·스타일 테이블 같은 이성 영역은 유저가 지시할 때만 포함하며, 유저 의도를 정확히 파악한 뒤 반영한다. 모호하면 넣지 않는다.

**유저 맞춤: 독해 난이도**
어휘 수준, 문장 복잡도, 전문 용어 밀도를 유저 수준에 맞춘다.

### 2. C단계 작성 기준

다음 세 가지는 사후 검토 기준이 아니라, C단계에서 결과물을 **작성하는 동안** 적용하는 판단 기준이다. 분량과 형식에 무관하게 적용된다. 해당 분야의 전문성만 갖추고 있다면, 낮은 인지 수준이더라도 무관하게 결과물을 온전히 납득하고 수용할 수 있어야 한다. 그만큼 빡빡하면서도 ai 특유의 허풍은 없어야 한다. 

**완결성**
전제, 비유, 질문, 비교를 열 때, 결과물 내에서 귀결시킬 수 있는 경우에만 연다. 귀결 계획 없이 여는 것은 금지한다. 작성 중 귀결이 불가능해진 아크는 삭제한다.

**추적 가능성**
아이디어 A에서 B로 전환할 때, 독자가 A→B의 근거를 본문 내에서 재구성할 수 있어야 한다. 전환을 쓰는 시점에 근거를 함께 배치한다. 근거를 배치할 수 없는 전환은 만들지 않는다.

**자족성**
본문이 외부 해설 없이 단독으로 기능하도록 쓴다. 메타 설명("이 글은~", "여기서 말하고자 하는 것은~")은 쓰지 않는다. 메타 설명이 필요하다고 느끼는 지점은 본문 서술 자체가 불충분하다는 신호이므로, 본문을 고친다.

**기대치 초과**
DIRECTION 내의 `[기대치 정의]` 블록을 읽고, 그 천장을 넘어서라. "이 정도면 됐다"가 아니라 "이건 예상 못 했다"가 반응이어야 한다. 기대치가 전문가 수준이면 전문가가 동료에게 공유할 만한 밀도를, 5세 기준이면 5세가 소리 지르며 좋아할 만한 것을 만들어라.

**수직적 깊이**
DIRECTION이 설정한 프레임 안에서만 판다. 밖으로 나가지 않는다. "RP 캐릭터 시트"를 요청받았으면 갑자기 소설이나 심리학 논문을 쓰지 않는다. "5세 기준"이라 했으면 20대 기준을 섞지 않는다. 해당 프레임 내부에서의 밀도와 정교함으로 승부한다.

## 🏅 기본 품질 키워드 (Default PREFERRED_IMAGERY)

유저가 별도로 지정하지 않아도 direction에 항상 포함하는 5개 키워드. 주제와 무관하게 결과물의 품질을 끌어올린다.

| 키워드 | 역할 |
|--------|------|
| 흐르는 결 | 추상을 구체적 질감으로 끌어내린다 |
| 응축된 밀도 | 정보 밀도를 높이고 허풍/미사여구를 압착한다 |
| 맨눈의 작동상 | "어떻게 돌아가는가"가 눈에 보이도록 강제한다 |
| 날선 접점 | 영역 간 충돌에 마찰과 긴장을 부여한다 |
| 무게감 | 수사가 뜨지 못하게 실감으로 잡아당긴다 |

유저가 PREFERRED_IMAGERY를 직접 지정한 경우, 유저의 키워드 뒤에 이 5개를 병합한다. 중복되는 개념이 있으면 유저 키워드를 우선한다.

## 💡 Core Concept

Stochastic Context Pollution:
- 깨끗한 맥락 → 확률적 평균 수렴 (진부한 결과)
- 오염된 맥락 → 창의적 연결 강제 (혁신적 결과)

LLM은 스스로를 놀라게 할 수 없다. 외부 무작위성(Python random.sample)이 확률 곡선을 깬다.
